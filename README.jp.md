# Chronicle

Chronicleは、自律型AIエージェントがワークフローの中で得た「経験」を、xAPI規格に基づいてLRS（Learning Record Store）に記録・参照可能にするためのオープンソース・プロジェクトです。

## 背景とビジョン

現在、自律型AIエージェントは高度な推論を通じて開発やタスクを遂行していますが、その試行錯誤のプロセスは個別のチャットログに埋もれがちです。プロンプトや実行結果を統一的に記録する標準的な手法も確立されていません。

Chronicleは、教育工学の標準規格である **xAPI** を採用し、エージェントの行動を「学習経験」として構造化します。これにより、エージェントは過去の「開発経験の記憶」を自ら参照して精度を高めたり、他のエージェントの成功・失敗事例を再現可能な記録として活用したりすることが可能になります。

## プロジェクトの目標

本プロジェクトでは、以下の3点の実現をロードマップとして掲げています。

1. **エージェント行動記録プロファイルの定義**: 自律型AIエージェント特有の振る舞いを記述するためのxAPIプロファイルの策定。
2. **記録用MCPプロキシの開発**: エージェントの行動をシームレスにLRSへ送信するためのMCP（Model Context Protocol）プロキシの実装。
3. **参照用MCPサーバーの開発**: 蓄積された経験データをエージェントが動的に取得・活用するためのインターフェースの提供。

将来的には、LRSに蓄積されたアクティビティを「ソフトウェアがどのように構築されたか」を証明するデータ（**SBOP: Software Bill of Process**）として品質保証に役立てたり、エージェントの経験を集約したグローバルなレジストリへと発展させることを目指します。

## 生成AIの活用について

本プロジェクトでは、コード実装や仕様策定のあらゆる場面で生成AIを積極的に活用しています。特に、深い考察を要するドキュメントは日本語で執筆し、生成AIを用いて英語へ翻訳するプロセスを採用しています。そのため、英文の多くはAIによる翻訳をベースとしています。
